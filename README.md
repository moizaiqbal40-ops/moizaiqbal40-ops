## How I Work

- **Document the failure modes, not just the wins** — MindEase's README explains exactly where its decision tree underperforms and why; that's a stronger signal than a suspiciously clean accuracy number
- **Match the tool to the domain** — the intrusion detection project exists because it mirrors an actual physical-security use case, not because YOLO was trending
- **OOP fundamentals matter as much as AI** — the task scheduler exists specifically to demonstrate abstraction and polymorphism cleanly, without ML as a distraction
- **Small, honest commits** — a Git history that reads like clear thinking
